<Info>

</Info>

## Setting Up Your Development Environment

Follow these steps to set up your local development environment:

<Steps titleSize="h3">

<Step title="Clone the Repository" icon="code-branch" iconType="solid">
Clone the repository and navigate to the project directory.

<CodeBlock>
```bash
git clone https://github.com/VRSEN/agency-swarm.git
cd agency-swarm
```
</CodeBlock>
</Step>

<Step title="Create a Virtual Environment" icon="laptop-code" iconType="solid">
Create and activate a virtual environment.

<CodeBlock>
```bash
python3 -m venv venv
source venv/bin/activate  # On Windows use `venv\Scripts\activate`
```
</CodeBlock>
</Step>

<Step title="Install Dependencies" icon="download" iconType="solid">
Install the required packages.

<CodeBlock>
```bash
pip install -r requirements-dev.txt
```
</CodeBlock>
</Step>

<Step title="Install Pre-Commit Hooks" icon="check-circle" iconType="solid">
Install pre-commit hooks for code quality checks.

<CodeBlock>
```bash
pip install pre-commit
pre-commit install
```
</CodeBlock>
</Step>

</Steps>

## Running Tests

<Note>
Testing ensures that your contributions work as intended and do not break existing functionality.
</Note>

<Steps titleSize="h3">

<Step title="Install Test Dependencies" icon="tools" iconType="solid">
Install the test dependencies.

<CodeBlock>
```bash
pip install -r requirements-dev.txt
```
</CodeBlock>
</Step>

<Step title="Run Tests" icon="play-circle" iconType="solid">
Run the test suite using Pytest.

<CodeBlock>
```bash
pytest
```
</CodeBlock>
</Step>

<Step title="Check Test Coverage" icon="chart-bar" iconType="solid">
Check the test coverage.

<CodeBlock>
```bash
pytest --cov=agency_swarm tests/
```
</CodeBlock>
</Step>

</Steps>

## Contributing Agents and Tools

<Steps titleSize="h3">

<Step title="Agents" icon="user-secret" iconType="solid">
Place new agents in the `agency_swarm/agents/` directory within an appropriate category.

<CodeBlock>
```
agency_swarm/
└── agents/
    └── your_agent_category/
        └── AgentName/
            ├── __init__.py
            ├── AgentName.py
            ├── instructions.md
            └── tools/
                ├── ToolName.py
                └── ...
```
</CodeBlock>
</Step>

<Step title="Tools" icon="wrench" iconType="solid">
Add new tools to the `agency_swarm/tools/` directory under a relevant category.

<CodeBlock>
```
agency_swarm/
└── tools/
    └── your_tool_category/
        ├── __init__.py
        ├── ToolName.py
        └── ...
```
</CodeBlock>
</Step>

</Steps>

## Adding Tests

<Note>
Testing ensures that your contributions work as intended and do not break existing functionality.
</Note>

- **Location**: Place tests in the `agency_swarm/tests/` directory.

<CodeBlock title="Example Test" language="python">
```python
def test_my_custom_tool():
    tool = MyCustomTool(example_field="test value")
    result = tool.run()
    assert "expected_output" in result
```
</CodeBlock>

## Submitting Changes

<AccordionGroup>

<Accordion title="1. Fork the Repository" icon="code-branch" iconType="solid">
Create a fork of the Agency Swarm repository on GitHub.
</Accordion>

<Accordion title="2. Create a Branch" icon="code" iconType="solid">
Work on a feature branch for your changes.

<CodeBlock language="bash">
```bash
git checkout -b feature/your-feature-name
```
</CodeBlock>
</Accordion>

<Accordion title="3. Submit a Pull Request" icon="paper-plane" iconType="solid">
Describe your changes and submit a pull request for review.
</Accordion>

</AccordionGroup>

## Guidelines

<Check>

- **Code Quality**: Follow PEP 8 style guidelines for Python code.

- **Documentation**: Update or add documentation for your contributions to help others understand your work.

- **Testing**: Ensure all new and existing tests pass before submitting your pull request.

</Check>
